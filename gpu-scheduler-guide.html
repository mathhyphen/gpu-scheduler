设计原则

GPU Scheduler 的核心假设是：你不会想让一个后台进程时刻连着你的 GPU 服务器。

它不会：

  • 在后台起 daemon 持续轮询 GPU 状态
  • 维持长连接心跳包
  • 在你没主动操作的时候访问服务器

它只会：

  • 你敲 list → 连上查一下 → 断开
  • 你敲 run → 连上查 GPU → 找到空闲 → 执行命令 → 断开
SSH 安全：因为不会频繁建立新连接，不会被 fail2ban 误判为爆破。 每次命令只做一次 SSH 握手，执行完就断。和你手动 ssh 上去跑命令的行为完全一致。

安装

# pipx 一键安装（推荐）
pipx install gpu-scheduler

# 或从源码安装
git clone https://github.com/mathhyphen/gpu-scheduler.git
cd gpu-scheduler && pip install -e .

配置服务器

1

生成配置文件

gpu-scheduler config init
2

编辑服务器列表

# ~/.config/gpu-scheduler/config.toml
[[servers]]
host = "gpu-server-1"
port = 22
user = "your-username"
key_file = "~/.ssh/id_rsa"

[[servers]]
host = "gpu-server-2"
port = 22
user = "your-username"
key_file = "~/.ssh/id_rsa"

[scheduler]
poll_interval = 5.0
3

测试连接

gpu-scheduler config test

配置文件按优先级搜索：$GPU_SCHEDULER_CONFIG → 当前目录 .gpu-scheduler.toml → ~/.config/gpu-scheduler/config.toml

日常命令

命令做什么连服务器？
gpu-scheduler list 查看所有 GPU 状态（显存、利用率、温度、进程数） 连一下，查完即断
gpu-scheduler run <cmd> 找空闲 GPU → SSH 执行命令 → 返回结果 连一下，跑完即断
gpu-scheduler config test 测试所有服务器的 SSH 连通性和 GPU 可用性 连一下，测完即断
gpu-scheduler config show 显示当前配置 不连
gpu-scheduler queue 查看任务队列（仅 daemon 模式） 不连

list — 查看 GPU 状态

# 一次性查询
gpu-scheduler list

# 实时刷新（每 3 秒查一次，Ctrl+C 停止）
gpu-scheduler list --watch

# 指定配置文件和刷新间隔
gpu-scheduler list -c .gpu-scheduler.toml -i 5 --watch

输出彩色表格：绿色=空闲，黄色=使用中，红色=满载。右侧显示每张卡的进程数。

run — 立即执行任务

# 1 张 GPU，默认显存要求
gpu-scheduler run python train.py

# 2 张 GPU
gpu-scheduler run python train.py --gpus 2

# 要求每张 GPU 至少 24GB 空闲显存
gpu-scheduler run python large_model.py --gpu-memory 24000

# 指定配置文件
gpu-scheduler run -c .gpu-scheduler.toml python infer.py --gpus 1
关于后台执行：run 会阻塞等待命令完成。 如果你的终端关了或 Ctrl+C，SSH 断开，远程进程会随之中止。 训练几小时的模型，请保持终端开着，或用 nohup：
gpu-scheduler run "nohup python train.py > train.log 2>&1 &"
这样即使断开，远程进程也会继续跑。

调度逻辑

每次 run 的执行流程：

1

并行 SSH 到所有配置的服务器，执行 nvidia-smi 查询

2

解析 GPU 信息（显存、利用率、进程），通过 UUID 精确映射进程到 GPU

3

筛选空闲 GPU（显存占用 < 500MB 且无进程）

4

在同一台服务器上找满足数量和显存要求的空闲卡，选显存最大的那些

5

SSH 到该服务器，设置 CUDA_VISIBLE_DEVICES，执行用户命令

6

命令完成，返回 exit code 和输出，断开连接

队列模式（可选）

如果你需要批量提交多个任务、按优先级排队执行：

# 终端 1：启动 daemon（会持续轮询，不推荐日常使用）
gpu-scheduler daemon

# 终端 2：提交任务
gpu-scheduler submit python train1.py --priority 0
gpu-scheduler submit python train2.py --priority 10
gpu-scheduler submit python infer.py --gpu-memory 32000

# 查看队列
gpu-scheduler queue

# 取消任务
gpu-scheduler cancel 3

daemon 模式下连接池会维持长连接。安全但不如 run 简洁。日常使用推荐 run。
